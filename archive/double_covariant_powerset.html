Self-Composing the Powerset Monad

For my first post, I wanted to give some explanation about the technical content of my PhD, but for a more general audience. In this post I give some informal intuition as to why there is no monad structure on the covariant powerset functor composed with itself. This informal reasoning is loosely from the viewpoint of modelling systems coalgebraically.

Motivation

A function \(f:X\to P(X)\), where \(P(X)\) is the powerset of \(X\), is a model for a nondeterministic system: at each state \(x\in X\) the subset \(f(x)\) gives the systems possible choices of states to proceed to. Composition of nondeterministic systems reflects our interpretation. We have an associative composition operator \( (;) \), which has a unit \(\eta\): $$ (f;g);h=f;(g;h)\hspace{3em} \eta;f=f\hspace{3em} f;\eta=f $$ In pictures, composition is given by a "glue" and a "flatten":

Formally, we define \(\eta:X\to P(X)\) as the "unit game" \(\eta(x):=\{x\}\), and composition relies heavily on set union: $$ f;g := (X\xrightarrow{f}P(X)\xrightarrow{P(g)}PP(X)\xrightarrow{\bigcup}PX)$$ Note that \((;)\) being associative and unital follow from properties of \(\bigcup\) and \(\eta\) This all gives, an admittedly simplistic, model of composable nondeterministic systems.

Two-Player Games

Now what if one wanted to model a game? Well, a natural choice would be to use of a set of subsets, rather than merely a set like for nondeterministic systems. Thus, we choose to interpret a function \(f:X\to PP(X)\) as a kind of two-player game, and, for convenience, name the players Angel and Demon. For each state \(x\in X\), \(f(x)\subseteq P(X)\) gives a set (of subsets) representing the choices available to the Angel. Each choice \(U\in f(x)\) will then give a set of states which Demon can resolve. These games are really from Angel's perspective, we give two drawing of a possible "view", from the perspective of Angel, from a state \(x\):
$$ f(x)=\{\{x_1,x_2\},\{x_3,x_4\},\{x_4,x_5\}\} $$

Now, how should we compose? Following our previous strategy, we are looking for a well-behaved function \({\sf flatten}:PPPP(X)\to PP(X)\), to be used in: $$ f;g:=(X\xrightarrow{f}PP(X)\xrightarrow{PP(g)}PPPP(X)\xrightarrow{\sf flatten} PP(X)) $$
